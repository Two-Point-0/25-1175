PDO + MySQL prepared stmts
// processor.php — handles BOTH GET & POST requests
<?php
$method = $_SERVER['REQUEST_METHOD']; // "GET" or "POST"
// fetch data from appropriate array
if ($method === 'POST') {
  $first = $_POST['first_name'] ?? '';
  $last = $_POST['last_name'] ?? '';
  $email = $_POST['email'] ?? '';
  $pwd = $_POST['password'] ?? '0000';
} else { // GET method (visible parameters)
  $first = $_GET['first_name'] ?? '';
  $last = $_GET['last_name'] ?? '';
  $email = $_GET['email'] ?? '';
  $pwd = $_GET['password'] ?? '0000';
  $source = $_GET['source_tag'] ?? 'direct'; // extra GET flavour
}

// ✨ database connection (creative credentials)
$host = 'localhost'; $dbname = 'hyper_nexus'; $user = 'root'; $pass = '';
try {
  $pdo = new PDO("mysql:host=$host;dbname=$dbname;charset=utf8mb4", $user, $pass);
  $hashed = password_hash($pwd, PASSWORD_BCRYPT);
  $stmt = $pdo->prepare("INSERT INTO users (first_name, last_name, email, password_hash, request_method, extra_meta) VALUES (?,?,?,?,?,?)");
  $extraMeta = ($method === 'GET') ? "source:".($source??'') : "post_mode";
  $stmt->execute([$first, $last, $email, $hashed, $method, $extraMeta]);
  echo "✅ [ $method ] Success! User $first inscribed in DB.";
} catch(PDOException $e) {
  echo "🌀 DB error: ".$e->getMessage();
}
?>
SQL SCHEMA (creative table) 
CREATE DATABASE hyper_nexus;
USE hyper_nexus;

CREATE TABLE users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    first_name VARCHAR(70) NOT NULL,
    last_name VARCHAR(70) NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    request_method ENUM('GET','POST') DEFAULT 'POST',
    extra_meta TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
creative GET+POST integration steps
  • This UI toggles form method & action="processor.php" dynamically.
  • In GET mode: all fields appear in URL (great for testing / bookmarking).
  • In POST mode: sensitive data hidden, password default "0000" still respected.
  • PHP script detects request method and reads from $_GET or $_POST accordingly.
  • Always hashes password (Argon2i/BCRYPT) before storing — no plaintext.
  • Bonus: the database logs which method was used, plus extra_meta for GET tags.
